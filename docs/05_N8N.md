# n8n Integration Guide

n8n acts as the **external event bridge** in this system.  
It does NOT produce content, run AI, or replace Python modules.  
Its only job: receive webhooks from external services → route to Python + Kit + Telegram.

---

## What n8n Does in This System

| Use Case | Trigger | n8n Action |
|----------|---------|------------|
| Gumroad sale | Customer buys a product | Tag buyer in Kit + notify Python + Telegram |
| Lead magnet download | Free product downloaded | Add subscriber to Kit + notify Python + Telegram |
| YouTube 48h metrics | 48h after video upload | Fetch metrics → update Sheets → alert if low |
| Shopify stock change (Phase 2.5+) | Stock drops below threshold | Trigger content production via Python |
| Form submission (Phase 3+) | Landing page form filled | Add to Kit + notify Python |

## What n8n Does NOT Do

- ❌ Produce videos, PDFs, or any content
- ❌ Run AI models (Ollama, Gemini, Claude)
- ❌ Replace `workflow_manager.py` or `scheduler.py`
- ❌ Store your business data (it only routes it)

## Import Workflows

1. n8n UI → top-right menu → **Import from file**
2. Import each file from `n8n_workflows/`:
   - `01_gumroad_sale.json` — Phase 1
   - `02_youtube_metrics.json` — Phase 2

---

## Set Environment Variables in n8n

1. n8n UI → Settings → **Variables**
2. Add:
   | Variable | Value |
   |----------|-------|
   | `KIT_API_SECRET` | your Kit API secret |
   | `TELEGRAM_CHAT_ID` | your Telegram chat ID |
   | `YOUTUBE_API_KEY` | YouTube Data API key |

## Workflow 01 — Gumroad Sale Setup

**In Gumroad:**
1. Settings → Advanced → Ping URL
2. Set: `http://YOUR_SERVER_IP:5678/webhook/gumroad-sale`
3. Save

**Test:**
```bash
curl -X POST http://YOUR_SERVER_IP:5678/webhook/gumroad-sale \
  -H "Content-Type: application/json" \
  -d '{"email":"test@test.com","product_name":"Test Guide","price":0}'
```
You should receive a Telegram message instantly.

## Workflow 02 — YouTube Metrics Setup

`youtube_uploader.py` automatically calls n8n after each upload.  
Nothing to configure — it's already wired in the code.

**Verify it's working:**
After uploading a test video, check n8n's execution log for workflow 02.  
You should see it waiting 48 hours.

## Export Workflows for Backup

n8n UI → Workflows → Select all → Export  
Save JSONs to `n8n_workflows/` folder.  
`github_backup.py` will commit them to GitHub automatically at 02:00.

---

## Adding a New n8n Workflow

1. Build workflow in n8n UI
2. Export as JSON
3. Save to `n8n_workflows/your_workflow.json`
4. Commit (or wait for daily auto-backup)

**Rule:** Keep all business logic in Python. Use n8n only to receive events and route them.  
If you find yourself adding conditional logic to n8n beyond simple routing, move it to Python.
